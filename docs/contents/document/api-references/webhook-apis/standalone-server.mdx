# Standalone Server Mode

The standalone server mode provides a dedicated webhook server focused solely on receiving and processing Slack events. This lightweight deployment option is ideal for microservices architectures and scenarios where you only need webhook functionality without MCP tools.

For detailed architectural information, see [Webhook Server Architecture](/dev/next/architecture/webhook-server-architecture#deployment-architecture).

## Endpoint Overview

**Base URL**: `http://your-server:port`
**Primary Endpoint**: `POST /slack/events`

## API Endpoints

### POST /slack/events

Primary webhook endpoint for receiving Slack events.

**Request Headers:**
```http
Content-Type: application/json
X-Slack-Signature: v0=<signature>
X-Slack-Request-Timestamp: <timestamp>
```

**Response:**
```http
HTTP/1.1 200 OK
Content-Type: application/json

{"status": "ok"}
```

## Command Line Usage

### Basic Commands

```bash
# Start standalone webhook server with defaults
python -m slack_mcp.webhook

# Explicit configuration
python -m slack_mcp.webhook \
  --host 0.0.0.0 \
  --port 3000 \
  --log-level INFO \
  --retry 3
```

### Environment-Specific Examples

```bash
# Development
python -m slack_mcp.webhook \
  --host 127.0.0.1 \
  --port 3000 \
  --log-level DEBUG \
  --env-file .env.development

# Production
python -m slack_mcp.webhook \
  --host 0.0.0.0 \
  --port 8080 \
  --log-level INFO \
  --env-file .env.production \
  --retry 5

# Cloud deployment
python -m slack_mcp.webhook \
  --host 0.0.0.0 \
  --port ${PORT} \
  --no-env-file
```

## Configuration Options

### CLI Arguments (Standalone Mode Only)

| Argument | Type | Default | Description |
|----------|------|---------|-------------|
| `--host` | `string` | `"0.0.0.0"` | Server bind address |
| `--port` | `int` | `3000` | Server port |
| `--log-level` | `string` | `"INFO"` | Logging verbosity |
| `--slack-token` | `string` | `None` | Override bot token |
| `--env-file` | `string` | `".env"` | Environment file path |
| `--no-env-file` | `flag` | `False` | Disable env file loading |
| `--retry` | `int` | `3` | Network retry attempts |

**Note**: The `--integrated`, `--mcp-transport`, and `--mcp-mount-path` options are **not** available in standalone mode.

### Environment Variables

**Required:**
```bash
export SLACK_SIGNING_SECRET="your_signing_secret_here"
export SLACK_BOT_TOKEN="xoxb-your-bot-token-here"
```

**Optional:**
```bash
export SLACK_EVENTS_TOPIC="slack_events_production"
export QUEUE_BACKEND_TYPE="redis"
export REDIS_URL="redis://localhost:6379/0"
export HOST="0.0.0.0"
export PORT="8080"
export LOG_LEVEL="INFO"
```

## Queue Backend Support

The standalone server publishes events to configurable queue backends:

**Redis:**
```bash
export QUEUE_BACKEND_TYPE="redis"
export REDIS_URL="redis://localhost:6379/0"
```

**In-Memory (Development):**
```bash
export QUEUE_BACKEND_TYPE="memory"
```

## Response Codes

| Status | Description |
|--------|-------------|
| `200 OK` | Event processed successfully |
| `400 Bad Request` | Invalid request format or headers |
| `401 Unauthorized` | Authentication failed |
| `500 Internal Server Error` | Server processing error |
